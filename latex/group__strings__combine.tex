Row-\/wise concatenates the given list of strings columns and returns a single strings column result. 

Each new string is created by concatenating the strings from the same row delimited by the separator provided.

Any row with a null entry will result in the corresponding output row to be null entry unless a narep string is specified to be used in its place.

If {\ttfamily separate\+\_\+nulls} is set to {\ttfamily NO} and {\ttfamily narep} is valid then separators are not added to the output between null elements. Otherwise, separators are always added if {\ttfamily narep} is valid.

More than one column must be specified in the input {\ttfamily strings\+\_\+columns} table.

Concatenates a list of strings columns using separators for each row and returns the result as a strings column. 

Each new string is created by concatenating the strings from the same row delimited by the row separator provided for that row. The following rules are applicable\+:


\begin{DoxyItemize}
\item If row separator for a given row is null, output column for that row is null, unless there is a valid {\ttfamily separator\+\_\+narep} 
\item The separator is applied between two output row values if the {\ttfamily separate\+\_\+nulls} is {\ttfamily YES} or only between valid rows if {\ttfamily separate\+\_\+nulls} is {\ttfamily NO}.
\item If {\ttfamily separator\+\_\+narep} and {\ttfamily col\+\_\+narep} are both valid, the output column is always non nullable
\end{DoxyItemize}

Each new string is created by concatenating the strings from the same row (same list element) delimited by the row separator provided in the {\ttfamily separators} strings column.

Any null row in the {\ttfamily separators} column will also result in a null output row unless a valid {\ttfamily separator\+\_\+narep} scalar is provided to be used in place of the null separators.
