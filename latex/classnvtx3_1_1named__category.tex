\subsubsection*{template$<$typename D = domain\+::global$>$\newline
class nvtx3\+::named\+\_\+category$<$ D $>$}
A {\ttfamily category} with an associated name string. 

Associates a {\ttfamily name} string with a category {\ttfamily id} to help differentiate among categories.

For any given category id {\ttfamily Id}, a {\ttfamily \doxylink{classnvtx3_1_1named__category}{named\+\_\+category}(Id, \"{}name\"{})} should only be constructed once and reused throughout an application. This can be done by either explicitly creating static {\ttfamily \doxylink{classnvtx3_1_1named__category}{named\+\_\+category}} objects, or using the {\ttfamily \doxylink{classnvtx3_1_1named__category_a0f7446a3381a38e2edc62e1cefe7333b}{named\+\_\+category\+::get}} construct on first use helper (recommended).

Creating two or more {\ttfamily \doxylink{classnvtx3_1_1named__category}{named\+\_\+category}} objects with the same value for {\ttfamily id} in the same domain results in undefined behavior.

Similarly, behavior is undefined when a {\ttfamily \doxylink{classnvtx3_1_1named__category}{named\+\_\+category}} and {\ttfamily category} share the same value of {\ttfamily id}.
