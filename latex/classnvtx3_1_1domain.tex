\doxysubsection{详细描述}
{\ttfamily domain}s allow for grouping NVTX events into a single scope to differentiate them from events in other {\ttfamily domain}s. 

By default, all NVTX constructs are placed in the \"{}global\"{} NVTX domain.

A custom {\ttfamily domain} may be used in order to differentiate a library\textquotesingle{}s or application\textquotesingle{}s NVTX events from other events.

{\ttfamily domain}s are expected to be long-\/lived and unique to a library or application. As such, it is assumed a domain\textquotesingle{}s name is known at compile time. Therefore, all NVTX constructs that can be associated with a domain require the domain to be specified via a {\itshape type} {\ttfamily Domain\+Name} passed as an explicit template parameter.

The type {\ttfamily \doxylink{structnvtx3_1_1domain_1_1global}{domain\+::global}} may be used to indicate that the global NVTX domain should be used.

None of the C++ NVTX constructs require the user to manually construct a {\ttfamily domain} object. Instead, if a custom domain is desired, the user is expected to define a type {\ttfamily Domain\+Name} that contains a member {\ttfamily Domain\+Name\+::name} which resolves to either a {\ttfamily char const\texorpdfstring{$\ast$}{*}} or {\ttfamily wchar\+\_\+t const\texorpdfstring{$\ast$}{*}}. The value of {\ttfamily Domain\+Name\+::name} is used to name and uniquely identify the custom domain.

Upon the first use of an NVTX construct associated with the type {\ttfamily Domain\+Name}, the \"{}construct on first use\"{} pattern is used to construct a function local static {\ttfamily domain} object. All future NVTX constructs associated with {\ttfamily Domain\+Type} will use a reference to the previously constructed {\ttfamily domain} object. See {\ttfamily \doxylink{classnvtx3_1_1domain_a2a452b895ec4192cf09c5ac98d35b591}{domain\+::get}}.

Returns reference to an instance of a function local static {\ttfamily domain} object. 

Uses the \"{}construct on first use\"{} idiom to safely ensure the {\ttfamily domain} object is initialized exactly once upon first invocation of {\ttfamily \doxylink{classnvtx3_1_1domain_a2a452b895ec4192cf09c5ac98d35b591}{domain\+::get}\texorpdfstring{$<$}{<}Domain\+Name\texorpdfstring{$>$}{>}()}. All following invocations will return a reference to the previously constructed {\ttfamily domain} object. See \href{https://isocpp.org/wiki/faq/ctors\#static-init-order-on-first-use}{\texttt{ https\+://isocpp.\+org/wiki/faq/ctors\#static-\/init-\/order-\/on-\/first-\/use}}

None of the constructs in this header require the user to directly invoke {\ttfamily \doxylink{classnvtx3_1_1domain_a2a452b895ec4192cf09c5ac98d35b591}{domain\+::get}}. It is automatically invoked when constructing objects like a {\ttfamily \doxylink{classnvtx3_1_1domain__thread__range}{domain\+\_\+thread\+\_\+range}} or {\ttfamily category}. Advanced users may wish to use {\ttfamily \doxylink{classnvtx3_1_1domain_a2a452b895ec4192cf09c5ac98d35b591}{domain\+::get}} for the convenience of the \"{}construct on first use\"{} idiom when using domains with their own use of the NVTX C API.

This function is threadsafe as of C++11. If two or more threads call {\ttfamily \doxylink{classnvtx3_1_1domain_a2a452b895ec4192cf09c5ac98d35b591}{domain\+::get}\texorpdfstring{$<$}{<}Domain\+Name\texorpdfstring{$>$}{>}} concurrently, exactly one of them is guaranteed to construct the {\ttfamily domain} object and the other(s) will receive a reference to the object after it is fully constructed.

The domain\textquotesingle{}s name is specified via the type {\ttfamily Domain\+Name} pass as an explicit template parameter. {\ttfamily Domain\+Name} is required to contain a member {\ttfamily Domain\+Name\+::name} that resolves to either a {\ttfamily char const\texorpdfstring{$\ast$}{*}} or {\ttfamily wchar\+\_\+t const\texorpdfstring{$\ast$}{*}}. The value of {\ttfamily Domain\+Name\+::name} is used to name and uniquely identify the {\ttfamily domain}.
