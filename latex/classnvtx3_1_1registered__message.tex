\subsubsection*{template$<$typename D = domain\+::global$>$\newline
class nvtx3\+::registered\+\_\+message$<$ D $>$}
A message registered with NVTX. 

Normally, associating a {\ttfamily message} with an NVTX event requires copying the contents of the message string. This may cause non-\/trivial overhead in highly performance sensitive regions of code.

message registration is an optimization to lower the overhead of associating a message with an NVTX event. Registering a message yields a handle that is inexpensive to copy that may be used in place of a message string.

A particular message should only be registered once and the handle reused throughout the rest of the application. This can be done by either explicitly creating static {\ttfamily \doxylink{classnvtx3_1_1registered__message}{registered\+\_\+message}} objects, or using the {\ttfamily \doxylink{classnvtx3_1_1registered__message_a8e587b148ec78a04dc18b5917e00022b}{registered\+\_\+message\+::get}} construct on first use helper (recommended).
