\subsubsection*{template$<$class D = domain\+::global$>$\newline
class nvtx3\+::domain\+\_\+thread\+\_\+range$<$ D $>$}
A RAII object for creating a NVTX range local to a thread within a domain. 

When constructed, begins a nested NVTX range on the calling thread in the specified domain. Upon destruction, ends the NVTX range.

Behavior is undefined if a {\ttfamily \doxylink{classnvtx3_1_1domain__thread__range}{domain\+\_\+thread\+\_\+range}} object is created/destroyed on different threads.

{\ttfamily \doxylink{classnvtx3_1_1domain__thread__range}{domain\+\_\+thread\+\_\+range}} is neither moveable nor copyable.

{\ttfamily \doxylink{classnvtx3_1_1domain__thread__range}{domain\+\_\+thread\+\_\+range}}s may be nested within other ranges.

The domain of the range is specified by the template type parameter {\ttfamily D}. By default, the {\ttfamily \doxylink{structnvtx3_1_1domain_1_1global}{domain\+::global}} is used, which scopes the range to the global NVTX domain. The convenience alias {\ttfamily thread\+\_\+range} is provided for ranges scoped to the global domain.

A custom domain can be defined by creating a type, {\ttfamily D}, with a static member {\ttfamily D\+::name} whose value is used to name the domain associated with {\ttfamily D}. {\ttfamily D\+::name} must resolve to either {\ttfamily char const\texorpdfstring{$\ast$}{*}} or {\ttfamily wchar\+\_\+t const\texorpdfstring{$\ast$}{*}}
