Performs grouped aggregations on the specified values. 

The values to aggregate and the aggregations to perform are specified in an {\ttfamily \doxylink{structcudf_1_1groupby_1_1aggregation__request}{aggregation\+\_\+request}}. Each request contains a {\ttfamily \doxylink{classcudf_1_1column__view}{column\+\_\+view}} of values to aggregate and a set of {\ttfamily aggregation}s to perform on those elements.

For each {\ttfamily aggregation} in a request, {\ttfamily values\mbox{[}i\mbox{]}} is aggregated with all other {\ttfamily values\mbox{[}j\mbox{]}} where rows {\ttfamily i} and {\ttfamily j} in {\ttfamily keys} are equivalent.

The {\ttfamily size()} of the request column must equal {\ttfamily keys.\+num\+\_\+rows()}.

For every {\ttfamily \doxylink{structcudf_1_1groupby_1_1aggregation__request}{aggregation\+\_\+request}} an {\ttfamily \doxylink{structcudf_1_1groupby_1_1aggregation__result}{aggregation\+\_\+result}} will be returned. The {\ttfamily \doxylink{structcudf_1_1groupby_1_1aggregation__result}{aggregation\+\_\+result}} holds the resulting column(s) for each requested aggregation on the {\ttfamily request}s values. The order of the columns in each result is the same order as was specified in the request.

The returned {\ttfamily table} contains the group labels for each group, i.\+e., the unique rows from {\ttfamily keys}. Element {\ttfamily i} across all aggregation results belongs to the group at row {\ttfamily i} in the group labels table.

The order of the rows in the group labels is arbitrary. Furthermore, successive {\ttfamily \doxylink{classcudf_1_1groupby_1_1groupby_a221be714878787abea34232da4c56e23}{groupby\+::aggregate}} calls may return results in different orders.

Performs grouped scans on the specified values. 

The values to aggregate and the aggregations to perform are specified in an {\ttfamily \doxylink{structcudf_1_1groupby_1_1aggregation__request}{aggregation\+\_\+request}}. Each request contains a {\ttfamily \doxylink{classcudf_1_1column__view}{column\+\_\+view}} of values to aggregate and a set of {\ttfamily aggregation}s to perform on those elements.

For each {\ttfamily aggregation} in a request, {\ttfamily values\mbox{[}i\mbox{]}} is scan aggregated with all previous {\ttfamily values\mbox{[}j\mbox{]}} where rows {\ttfamily i} and {\ttfamily j} in {\ttfamily keys} are equivalent.

The {\ttfamily size()} of the request column must equal {\ttfamily keys.\+num\+\_\+rows()}.

For every {\ttfamily \doxylink{structcudf_1_1groupby_1_1aggregation__request}{aggregation\+\_\+request}} an {\ttfamily \doxylink{structcudf_1_1groupby_1_1aggregation__result}{aggregation\+\_\+result}} will be returned. The {\ttfamily \doxylink{structcudf_1_1groupby_1_1aggregation__result}{aggregation\+\_\+result}} holds the resulting column(s) for each requested aggregation on the {\ttfamily request}s values. The order of the columns in each result is the same order as was specified in the request.

The returned {\ttfamily table} contains the group labels for each row, i.\+e., the {\ttfamily keys} given to groupby object. Element {\ttfamily i} across all aggregation results belongs to the group at row {\ttfamily i} in the group labels table.

The order of the rows in the group labels is arbitrary. Furthermore, successive {\ttfamily \doxylink{classcudf_1_1groupby_1_1groupby_a695b525d3874c8190f9d3d9ad03c8900}{groupby\+::scan}} calls may return results in different orders.
