\doxysubsection{详细描述}
A non-\/owning, immutable view of device data as a column of elements, some of which may be null as indicated by a bitmask. 

A {\ttfamily \doxylink{classcudf_1_1detail_1_1column__view__base}{column\+\_\+view\+\_\+base}} can be constructed implicitly from a {\ttfamily \doxylink{classcudf_1_1column}{cudf\+::column}}, or may be constructed explicitly from a pointer to pre-\/existing device memory.

Unless otherwise noted, the memory layout of the {\ttfamily \doxylink{classcudf_1_1detail_1_1column__view__base}{column\+\_\+view\+\_\+base}}\textquotesingle{}s data and bitmask is expected to adhere to the Arrow Physical Memory Layout Specification\+: \href{https://arrow.apache.org/docs/memory_layout.html}{\texttt{ https\+://arrow.\+apache.\+org/docs/memory\+\_\+layout.\+html}}

Because {\ttfamily \doxylink{classcudf_1_1detail_1_1column__view__base}{column\+\_\+view\+\_\+base}} is non-\/owning, no device memory is allocated nor freed when {\ttfamily \doxylink{classcudf_1_1detail_1_1column__view__base}{column\+\_\+view\+\_\+base}} objects are created or destroyed.

To enable zero-\/copy slicing, a {\ttfamily \doxylink{classcudf_1_1detail_1_1column__view__base}{column\+\_\+view\+\_\+base}} has an {\ttfamily offset} that indicates the index of the first element in the column relative to the base device memory allocation. By default, {\ttfamily \doxylink{classcudf_1_1detail_1_1column__view__base_a8bf33063791c751052a8ef20cf20468c}{offset()}} is zero.
